# Domain Model: Combo Portfolio

---

## 1. Bounded Context

**Context Name:** Combo Portfolio Context

**Responsibility:** Owns the lifecycle of saved combos — creation, naming, sharing, visibility management, and deletion. It is the single source of truth for all persisted combo data. It does not make styling decisions (that belongs to the AI Styling Engine context) but it enriches saved item snapshots with live product data on read.

**Bounded Context Boundary:**
- **Upstream dependency (read enrichment):** Platform Product Catalog (Unit 1) — called by the `ComboEnrichmentService` to refresh item details.
- **Downstream consumer:** Cart Handoff Context (Unit 5) — consumes this context via the `GET /api/v1/combos/{id}` API.
- **Public-facing read:** An unauthenticated public endpoint exposes combo data identified by a `ShareToken`.

---

## 2. Ubiquitous Language

| Term | Definition |
|---|---|
| **Combo** | A named, persisted collection of 2–10 fashion items saved by a shopper, originally sourced from an AI styling suggestion. |
| **Shopper** | An authenticated user who owns, manages, and may share their combos. |
| **Combo Item** | A snapshot reference to a product at the time the combo was saved, identified by `configSku` and `simpleSku`. Snapshot data (name, imageUrl, price) is stored at save time but enriched with live catalog data on read. |
| **Visibility** | The access level of a combo — `public` (anyone with the link can view) or `private` (owner only). |
| **Share Token** | A unique, opaque identifier appended to the public share URL. Present only when the combo has been explicitly shared. Revoked when visibility is set to private. |
| **Enriched Combo** | A read-only projection of a Combo where each Combo Item's data has been refreshed from the product catalog. Used for display purposes. |
| **Save Combo** | The action of persisting a new combo with a name and item list. |
| **Rename** | The action of changing a combo's name without altering its items. |
| **Share** | The action of generating a Share Token and making the combo publicly accessible. |
| **Make Private** | The action of setting visibility to `private`, which simultaneously revokes the Share Token. |

---

## 3. Aggregates

### Aggregate Root: `Combo`

The `Combo` aggregate is the central model of this bounded context. It encapsulates all state and enforces all invariants related to a saved combo's lifecycle.

**Invariants enforced by the aggregate:**
- A combo must contain between 2 and 10 `ComboItem`s (inclusive). Any command that would violate this is rejected.
- A `ShareToken` may only exist when `Visibility` is `public`.
- When `Visibility` is changed to `private`, the `ShareToken` is immediately nullified within the same aggregate operation.
- A `ShareToken` is only generated by an explicit Share command — not automatically on creation.
- A combo always belongs to exactly one `ShopperId`.

---

## 4. Entities

There are no subordinate entities within the `Combo` aggregate beyond the aggregate root itself. All inner data is modelled as Value Objects.

---

## 5. Value Objects

### `ComboId`
- **Type:** UUID string
- **Constraints:** Immutable after creation; globally unique.
- **Description:** The identity of a Combo aggregate.

---

### `ShopperId`
- **Type:** String
- **Constraints:** Non-empty; sourced from the authentication context. Immutable.
- **Description:** Identifies the shopper who owns the combo. Not validated internally — treated as an opaque identifier from the auth layer.

---

### `ComboName`
- **Type:** String
- **Constraints:** Non-empty. Maximum 100 characters. Leading/trailing whitespace is trimmed.
- **Description:** The human-readable name given to a combo by the shopper. A default name is suggested at save time (e.g., "Combo – March 31, 2026") but must be supplied by the client.

---

### `ComboItem`
- **Fields:**
  - `configSku` — String; the product's configurable SKU (identifies the product line)
  - `simpleSku` — String; the product's specific variant SKU (identifies exact size/color variant)
  - `name` — String; product name snapshot at save time
  - `imageUrl` — String; product image URL snapshot at save time
  - `price` — Decimal; product price snapshot at save time
- **Constraints:** All fields are required and non-empty. Two `ComboItem`s are considered equal if their `simpleSku` values match. A combo must not contain duplicate `simpleSku`s.
- **Note on enrichment:** `name`, `imageUrl`, and `price` are stored as snapshots. On read, the `ComboEnrichmentService` refreshes these fields from the product catalog. The snapshot serves as a fallback if the product is no longer available in the catalog.

---

### `ShareToken`
- **Type:** UUID string, or absent (null)
- **Constraints:** When present, must be globally unique. Generated by the `ShareTokenGenerationService`. Nullified (not deleted) when visibility changes to private.
- **Description:** The opaque token embedded in the public share URL. Its presence implies the combo has been shared at least once. Its absence means the combo has never been shared or was made private.

---

### `Visibility`
- **Type:** Enum — `public` | `private`
- **Default:** `private` (all combos are private at creation)
- **Constraints:** Changing to `private` must simultaneously nullify `ShareToken` within the aggregate. Changing to `public` alone does not generate a `ShareToken`; an explicit Share command is required.

---

## 6. Domain Events

All domain events are immutable records of what happened within the aggregate. They carry the minimum data needed for downstream consumers.

### `ComboCreated`
- **Trigger:** A new Combo aggregate is successfully saved.
- **Payload:** `comboId`, `shopperId`, `comboName`, `itemCount`, `createdAt`
- **Consumers:** Audit log, analytics.

---

### `ComboRenamed`
- **Trigger:** The shopper changes the combo's name.
- **Payload:** `comboId`, `shopperId`, `oldName`, `newName`, `renamedAt`
- **Consumers:** Audit log.

---

### `ComboDeleted`
- **Trigger:** The shopper deletes the combo.
- **Payload:** `comboId`, `shopperId`, `deletedAt`
- **Consumers:** Audit log; any cached or shared links for this combo should be treated as expired.

---

### `ComboShared`
- **Trigger:** The shopper requests a share token for the combo. Visibility is set to `public` and a `ShareToken` is generated.
- **Payload:** `comboId`, `shopperId`, `shareToken`, `sharedAt`
- **Consumers:** Audit log, analytics (share engagement tracking).

---

### `ComboMadePrivate`
- **Trigger:** The shopper changes the combo's visibility to `private`. The `ShareToken` is revoked within the same operation.
- **Payload:** `comboId`, `shopperId`, `revokedShareToken`, `madePrivateAt`
- **Consumers:** Audit log; downstream systems relying on the share URL should invalidate their caches.

---

## 7. Policies

Policies are business rules that react to domain events or enforce cross-cutting invariants. They are enforced within the aggregate's command handlers unless noted otherwise.

### `ShareTokenRevocationPolicy`
- **Reacts to:** `ComboMadePrivate`
- **Rule:** When a combo's visibility is changed to `private`, the Share Token is immediately nullified. No separate command is needed — this is an atomic part of the Make Private operation.
- **Enforced by:** Aggregate root `Combo`, within the `makePrivate()` command handler.

---

### `ComboItemCountPolicy`
- **Reacts to:** Any command that sets or modifies the item list (Save Combo)
- **Rule:** The item list must contain at minimum 2 items and at most 10 items. A combo with fewer than 2 or more than 10 items is rejected before persistence.
- **Enforced by:** Aggregate root `Combo`, within the `create()` command handler.

---

### `UniqueItemPolicy`
- **Reacts to:** Any command that sets the item list
- **Rule:** No two `ComboItem`s in the same combo may share the same `simpleSku`.
- **Enforced by:** Aggregate root `Combo`, within the `create()` command handler.

---

### `ShareTokenVisibilityConsistencyPolicy`
- **Rule:** A `ShareToken` may only exist if `Visibility` is `public`. This invariant is enforced structurally — the `makePrivate()` handler always nullifies the token atomically.
- **Enforced by:** Aggregate root `Combo`.

---

## 8. Repositories

Repositories provide the persistence abstraction for the `Combo` aggregate. The domain layer depends on the repository interface; the infrastructure layer provides the implementation.

### `ComboRepository`

| Method | Description |
|---|---|
| `save(combo: Combo)` | Persists a new or updated Combo aggregate (upsert semantics). |
| `findById(id: ComboId): Combo \| null` | Retrieves a combo by its identity. Returns null if not found. |
| `findByShopperId(shopperId: ShopperId): Combo[]` | Returns all combos owned by a shopper, ordered by `createdAt` descending. |
| `findByShareToken(token: ShareToken): Combo \| null` | Retrieves a combo by its public share token. Returns null if no match or token has been revoked. |
| `delete(id: ComboId): void` | Removes the combo from persistence. |

---

## 9. Domain Services

### `ComboEnrichmentService`
- **Purpose:** Enriches the `ComboItem` snapshots stored in a `Combo` with live product data from the product catalog (Unit 1 / Platform Product API).
- **Why a Domain Service:** Enrichment requires calling an external bounded context (Product Catalog). This cross-context call cannot belong to the `Combo` aggregate itself, which must remain isolated from infrastructure.
- **Inputs:** A `Combo` aggregate (or a list of `ComboItem` value objects)
- **Outputs:** A read-model projection — an `EnrichedCombo` — containing the combo metadata alongside refreshed `name`, `imageUrl`, and `price` for each item. Availability (in-stock status) is also included in the enriched view.
- **Fallback behaviour:** If the product catalog returns no data for a given `simpleSku` (product removed from catalog), the service falls back to the stored snapshot values and marks the item as `catalogUnavailable: true`.
- **Note:** This service is used for read operations only. It does not mutate the `Combo` aggregate.

---

### `ShareTokenGenerationService`
- **Purpose:** Generates a globally unique `ShareToken` for a combo on explicit Share command.
- **Why a Domain Service:** Uniqueness of the token may require a check against the repository (to avoid collisions), which cannot be done by the aggregate in isolation.
- **Inputs:** A `ComboId`
- **Outputs:** A new `ShareToken` value object
- **Behaviour:** Generates a UUID-based token, verifies uniqueness via `ComboRepository.findByShareToken()`, and regenerates on collision (expected to be negligible in practice).

---

## 10. Context Map

| Relationship | Direction | Pattern |
|---|---|---|
| Combo Portfolio ↔ Platform Product Catalog | Outbound (read) | **Anticorruption Layer** — `ComboEnrichmentService` translates platform product API responses into domain `EnrichedComboItem` structures. |
| Combo Portfolio ↔ Cart Handoff Context | Inbound (consumed) | **Open Host Service / Published Language** — exposes `GET /api/v1/combos/{id}` as a stable API. |
| Combo Portfolio ↔ Auth Context | Inbound (trusted) | **Conformist** — `ShopperId` and authentication tokens are accepted as-is from the auth layer; no translation needed. |
